Honeypot AP Attack

  • Axel Corradine Marenco (79878)
  • Johan Sebastian Espino Riveros (128052)

Abstract

The honeypot access point attack is a cybersecurity technique in which a Wi-Fi access point is created, emitting a fake SSID that simulates being legitimate to make its victims connect to it. This attack aims to get users to connect, allowing the attacker to capture data or insert malware into the connected devices. The honeypot network is based on the concept of a decoy that attracts the victims' attention, making them believe they are in a safe environment.

Glosario

1. Ataque de punto de acceso de honeypot
Técnica en ciberseguridad que crea un punto de acceso Wi-Fi falso para atraer a las víctimas y capturar datos o insertar malware.
2. SSID (Service Set Identifier)
Nombre de una red Wi-Fi que se muestra a los dispositivos cercanos.
3. Captura de datos
Proceso mediante el cual un atacante intercepta y almacena información transmitida a través de la red.
4. Malware
Software malicioso diseñado para dañar, interrumpir o tomar el control de sistemas informáticos.
5. Señuelo
Estrategia utilizada para atraer a las víctimas, haciéndoles creer que están en un entorno seguro.
6. Intercepción de tráfico
Técnica que permite a un atacante monitorear y capturar datos transmitidos entre dispositivos conectados a la red.
7. Inyección de malware
Proceso de introducir software malicioso en los dispositivos conectados a la red.
8. Redes abiertas
Redes Wi-Fi que no requieren autenticación para conectarse, lo que las hace vulnerables a ataques.
9. VPN (Virtual Private Network)
ecnología que crea una conexión segura y cifrada a través de una red menos segura, como Internet.
10. Conexión automática
Configuración de dispositivos que permite conectarse automáticamente a redes Wi-Fi conocidas o abiertas.

¿En que consiste?

El ataque consiste en que el atacante configura una red Wi-Fi con un nombre conocido o familiar (SSID), como "Wi-Fi_Público", "Wifi Gratis," o cualquier nombre siempre que la red no tenga contraseña y se encuentre abierta, o con la clave visible, aunque esta opción es menos factible para este tipo de ataques. Una vez conectadas, el atacante puede interceptar el tráfico de la red, capturando contraseñas, sitios web visitados, datos bancarios y otra información sensible. Este tipo de red también puede usarse para instalar software malicioso en los dispositivos de los usuarios.

Carasteristicas

  1. Falsa legitimidad: La red Wi-Fi se presenta como una opción confiable, imitando nombres comunes para parecer legítima.
  2. Captura de tráfico: Permite al atacante monitorear el tráfico de los dispositivos conectados, accediendo a datos y contraseñas.
  3. Inyección de malware: Puede facilitar la instalación de malware en los dispositivos.
  4. Ocultamiento: Al simular un nombre de red común, muchos usuarios no sospechan de la trampa.
  5. Flexibilidad del ataque: El atacante puede decidir entre capturar datos, realizar ataques de "man-in-themiddle" o infectar dispositivos.

Escenarios

  1. Áreas públicas con Wi-Fi gratuito, cafés, aeropuertos, y centros comerciales, donde los usuarios tienden a conectarse a redes abiertas sin verificarlas.
  2. Zonas donde la gente espera redes conocidas, por ejemplo, cerca de un hotel o una cadena de tiendas, donde los usuarios creen que la red es del lugar.
  3. Redes corporativas donde el atacante simula una red similar a la de la empresa para engañar a empleados y obtener acceso a información sensible

Controles Proactivos

  1. Verificación de redes: En lugares públicos, preguntar al personal para confirmar el nombre de la red antes de conectarse.
  2. Uso de VPN: Una VPN cifra el tráfico, haciendo que, incluso en una red falsa, los datos del usuario sean difíciles de interceptar.
  3. Desactivar la conexión automática a redes abiertas: Esto permite al usuario verificar cada red antes de conectarse.
  4. Mantener el dispositivo actualizado: Las actualizaciones de seguridad protegen contra posibles vulnerabilidades.

¿Como se contrarestan?

  1. Bloqueo de conexiones automáticas: Configurar dispositivos para que no se conecten automáticamente a redes Wi-Fi abiertas.
  2. Monitorización y análisis de red: Los sistemas de seguridad de redes pueden detectar patrones anómalos en las conexiones Wi-Fi y alertar de posibles redes honeypot.
  3. Concienciación del usuario: Educar a los usuarios sobre los riesgos de conectarse a redes abiertas desconocidas y cómo identificar posibles honeypots.
  4. Utilización de soluciones de seguridad de red: Los sistemas avanzados de seguridad en redes pueden identificar ataques de honeypot y mitigarlos.

Pregunta Parcial

¿Cuál es el objetivo principal del ataque de punto de acceso de honeypot?

  • A) Capturar datos o insertar malware en los dispositivos conectados.
  • B) Mejorar la velocidad de la red Wi-Fi.
  • C) Proporcionar acceso gratuito a Internet.
  • D) Aumentar la cobertura de la red Wi-Fi.

Respuesta correcta:
A) Capturar datos o insertar malware en los dispositivos conectados.
